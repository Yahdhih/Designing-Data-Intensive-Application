### Schéma 2 — Diagramme de séquence : 6 anomalies (20 min)

Dessine un diagramme de séquence pour **chacune des 6 anomalies** (format : T1 et T2 en colonnes) :

**1. Dirty read**
```
T1                    T2
BEGIN                 BEGIN
                      UPDATE x = 50 (pas encore commité)
SELECT x → 50 ← dirty read !
                      ROLLBACK (x revient à 100)
```

**2. Non-repeatable read**
```
T1                    T2
BEGIN (SI = Read Committed)
SELECT x → 100
                      BEGIN
                      UPDATE x = 50
                      COMMIT
SELECT x → 50 ← différent ! (même txn, même requête)
```

**3. Lost update**
```
T1                    T2
BEGIN
v = SELECT x → 100
                      BEGIN
                      v = SELECT x → 100
UPDATE x = v + 10 → 110
COMMIT
                      UPDATE x = v + 20 → 120 ← écrase T1 !
                      COMMIT
Final : 120 (attendu 130)
```

**4. Write skew (médecins)**
```
T1 (Alice)            T2 (Bob)
BEGIN SI
n = COUNT(*) → 2
                      BEGIN SI
                      n = COUNT(*) → 2
UPDATE alice = OFF
COMMIT
                      UPDATE bob = OFF
                      COMMIT
Final : 0 médecins (invariant violé)
```

**5. Phantom read**
```
T1                    T2
BEGIN REPEATABLE READ
SELECT * WHERE date=X → 0 résultats
                      BEGIN
                      INSERT (date=X, ...)
                      COMMIT
INSERT (date=X, ...) → succès (double réservation !)
COMMIT
```

**6. Dirty write**
```
T1                    T2
BEGIN
UPDATE x = "Alice"    BEGIN
(pas encore commité)  UPDATE x = "Bob" ← écrase Alice !
                      COMMIT
ROLLBACK
Final : "Bob" (mais x était "Alice" quand T2 a écrit)
```

### Schéma 3 — Solutions par anomalie (10 min)

Dessine un tableau de solutions :

| Anomalie | Solution 1 (SQL) | Solution 2 (niveau d'isolation) |
|----------|-----------------|--------------------------------|
| Dirty read | — | Read Committed |
| Non-repeatable read | — | REPEATABLE READ |
| Lost update | `UPDATE x = x + n` (atomique) | REPEATABLE READ ou FOR UPDATE |
| Write skew | `SELECT ... FOR UPDATE` | SERIALIZABLE |
| Phantom read | Index unique + contrainte | SERIALIZABLE |
| Dirty write | — | Read Committed (row-level lock) |
