## 📝 Mini-quiz Ch.7 final — 20 min

> Les 10 questions les plus importantes de Ch.7.

**1.** Un système de paiement débite un compte et crédite un autre. Sans transaction, qu'est-ce qui peut mal se passer ? Quelle propriété ACID résout chaque problème potentiel ?

**2.** Un ingénieur te dit : « On n'a pas besoin de transactions — on a des retries dans le client. » Donne un exemple où les retries seuls ne suffisent pas à remplacer l'atomicité.

**3.** Postgres utilise MVCC. Si une transaction longue (1 heure) maintient un snapshot, quel problème peut survenir pour Postgres en arrière-plan ? (Indice : `VACUUM`)

**4.** Deux transactions T1 et T2 veulent chacune incrémenter le même compteur. Avec Read Committed, quelle est la valeur finale si le point de départ est 0 et les 2 font `v = SELECT; UPDATE counter = v + 1` ? Avec `UPDATE counter = counter + 1` ? Justifie.

**5.** Quel est le comportement d'un `SELECT FOR UPDATE` quand la ligne ciblée est déjà verrouillée par une autre transaction ? Donne les 3 options SQL et leur comportement.

**6.** Un système utilise `ISOLATION LEVEL SERIALIZABLE`. Sous haute contention (1000 req/s sur les mêmes données), que se passe-t-il au niveau des performances vs 2PL traditionnel ? (SSI vs 2PL)

**7.** Write skew vs phantom read : donne un exemple où seul Serializable suffit (ni FOR UPDATE ni Repeatable Read) — et explique pourquoi.

**8.** Un ingénieur veut utiliser `BEGIN ISOLATION LEVEL SERIALIZABLE` pour toutes les transactions d'une application. Quels changements d'application sont nécessaires pour que ça fonctionne correctement en production ?

**9.** MongoDB ajoute des transactions multi-documents depuis la version 4.0. Quel niveau d'isolation offrent-elles par défaut ? Pourquoi est-ce différent du comportement historique de MongoDB (document-level atomicity) ?

**10.** Cassandra n'a pas de transactions multi-lignes. Comment un ingénieur peut-il implémenter une opération « transférer N € de A vers B » sur Cassandra sans perdre de l'argent ? Quel est le compromis ?
