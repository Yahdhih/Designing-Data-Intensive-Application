### Schéma 1 — Réplication mono-leader standard (7 min)

Dessine dans `reference/notes/jour-041-schemas.md` (ou sur papier) :

```
Contraintes à respecter dans ton dessin :
  - 1 Leader, 2 Followers
  - Chemin d'une écriture : client → leader → followers (sync ET async)
  - Chemin d'une lecture : client → follower (avec lag possible)
  - Où est le WAL ? Le log de réplication ?
  - Failover : qui devient leader quand le leader tombe ?
```

Note à côté : les 3 anomalies de lag (RYW, monotonic, prefix) avec 1 phrase chacune.

### Schéma 2 — Réplication mono-leader avec lecture depuis followers (7 min)

```
Contraintes :
  - 1 Leader (écritures) + 3 Followers (lectures scale-out)
  - Indiquer le lag de réplication asynchrone
  - Montrer quelle garantie (RYW) routing vers le leader ou follower
  - Indiquer le point de défaillance (SPOF) : le leader
```

### Schéma 3 — Réplication multi-leader multi-datacenter (7 min)

```
Contraintes :
  - 2 datacenters, 1 leader + 1 follower par datacenter
  - Les leaders se répliquent mutuellement (asynchrone cross-DC)
  - Les followers reçoivent du leader *local*
  - Montrer où le conflit peut survenir
  - Indiquer la résolution : LWW ou CRDT ou app
```

### Schéma 4 — Réplication leaderless (Dynamo-style) avec N=5, W=3, R=3 (7 min)

```
Contraintes :
  - 5 nœuds disposés en anneau (consistent hash)
  - Une écriture : coordinateur → W=3 nœuds → succès si 3 confirment
  - Une lecture : coordinateur → R=3 nœuds → valeur avec version max
  - Montrer W+R>N = 6 > 5 → intersection ≥ 1
  - Montrer le hinted handoff si 1 nœud est DOWN
```

### Schéma 5 — Vector clocks / détection de concurrence (6 min)

```
Exemple avec 3 clients (A, B, C) et 1 nœud :
  - Écriture 1 : A écrit v1 = "noir"   → clock: {A:1}
  - Écriture 2 : B écrit v2 = "blanc"  → clock: {B:1}  (concurrent avec A !)
  - Écriture 3 : C lit {v1, v2} et fusionne → v3 = "gris" → clock: {A:1, B:1, C:1}

Questions à répondre sur le schéma :
  - Comment sait-on que v1 et v2 sont concurrentes (pas causalement ordonnées) ?
  - Comment sait-on que v3 "domine" v1 et v2 ?
```
