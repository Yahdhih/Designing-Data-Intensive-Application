### Points clés à retenir

1. **Backward compatibility** (compatibilité descendante) :
   - **Nouveau code peut lire les vieux données** (déjà sur disque ou dans des anciens messages)
   - Plus facile : on sait ce que les anciens données contiennent
   - Ajout d'un champ optionnel → backward compat automatique (vieux code l'ignore)

2. **Forward compatibility** (compatibilité ascendante) :
   - **Vieux code peut lire les nouvelles données**
   - Plus difficile : le vieux code ne sait pas encore ce que le nouveau format contient
   - Vieux code **ignore** les champs inconnus → nécessite que le décodeur soit tolérant

3. **Rolling upgrade = les deux en même temps :**
   - Pendant un déploiement progressif, v1 et v2 du service coexistent
   - v1 écrit des messages que v2 doit lire (backward)
   - v2 écrit des messages que v1 doit lire (forward)
   - → Le format doit être compatible dans les **deux sens**

4. **3 modes de dataflow :**
   - **Via base de données :** `UPDATE` = écriture par un processus, lecture par un autre. Si v1 et v2 lisent la même table → même schéma doit servir les deux.
   - **Via services (REST/RPC) :** client appelle le serveur. Si le client est v1 et le serveur v2 → forward compat du serveur. Client peut être mis à jour plus lentement.
   - **Via message queue (Kafka, etc.) :** producteur et consommateur peuvent être mis à jour indépendamment, à des rythmes différents. Le schéma du message doit être stable.

5. **Kafka et compatibilité :**
   - Dans Kafka, les messages restent dans le log pendant des jours/semaines
   - Un consommateur peut rejouer des messages anciens (backward compat)
   - Un producteur peut envoyer des messages à un consommateur pas encore mis à jour (forward compat)
   - → Avro + Schema Registry gère ça avec `BACKWARD`, `FORWARD`, `FULL` compatibility modes

### 📌 Contexte 2026 (si 1ʳᵉ éd.)
La 2ᵉ éd. couvre davantage **gRPC** (Protobuf + HTTP/2) qui est devenu le standard de facto pour la communication inter-services. Le principe de compatibilité reste identique mais les outils ont évolué.
